# 架构决策记录（ADR）

这里记录会影响长期维护的重要技术选择。小的 UI 调整不需要写 ADR；涉及发布、安全、数据源、自动更新、长期依赖的决策应记录。

## 模板

```markdown
## ADR-001: [决策标题]

**日期**: YYYY-MM-DD
**状态**: 提议 / 已接受 / 已废弃

### 背景

为什么需要做这个决策？

### 决策

我们选择了什么？

### 后果

- 好处：
- 代价：
- 后续注意事项：
```
